# Phase 92 — Visual QA Screenshot Review Prep

این مرحله برای آماده‌سازی QA تصویری بعد از redesignهای اصلی است. در این مرحله هیچ فایل اجرایی UI/JSX/CSS تغییر داده نشده و فقط مستندات و چک‌لیست تست اضافه شده است.

## هدف

برای اینکه مرحله بعد بر اساس اسکرین‌شات واقعی دقیق و کم‌ریسک باشد، باید از هر صفحه در چند عرض مشخص عکس گرفته شود و مشکل‌ها با اولویت ثبت شوند.

## ترتیب گرفتن اسکرین‌شات

1. Settings > Telegram
2. Dashboard
3. Reports Hub
4. Financial Overview / Cashflow
5. Mobile Phones
6. Partner Detail
7. Customer Detail
8. Products
9. Services
10. Repairs
11. Settings > Account / Business / Users

## عرض‌های پیشنهادی

- 1366px دسکتاپ استاندارد
- 1280px لپ‌تاپ کوچک
- 768px تبلت
- 390px موبایل

## حالت‌های لازم

برای هر صفحه حداقل این دو حالت را تست کن:

- Light mode
- Dark mode

برای صفحات عملیاتی، یک اسکرین‌شات هم از حالت دارای داده زیاد بگیر؛ مثلاً جدول پر، لاگ زیاد، یا کارت‌های زیاد.

## قالب نام‌گذاری فایل‌ها

```txt
01-telegram-1366-light.png
02-telegram-1366-dark.png
03-dashboard-1280-light.png
04-mobilephones-390-dark.png
```

## چیزهایی که باید در هر تصویر بررسی شوند

- متن زیر آیکون نرفته باشد.
- دکمه‌ها روی هم نیفتاده باشند.
- عددها، IMEI، Chat ID و شناسه‌های فنی LTR درست باشند.
- کارت‌ها ارتفاع و spacing منطقی داشته باشند.
- header یا toolbar بیش از حد بزرگ نباشد.
- در dark mode کنتراست کافی باشد.
- در 1280px جدول‌ها نشکنند.
- در موبایل actionها قابل لمس و خوانا باشند.

## خروجی پیشنهادی برای مرحله بعد

از اولین صفحه یعنی Settings > Telegram شروع کن و ۲ اسکرین‌شات بفرست:

- 1366 Light
- 1366 Dark

بعد از اصلاح آن، سراغ داشبورد می‌رویم.
